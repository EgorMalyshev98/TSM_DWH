# TSM DWH

dbt-проект для DWH на основе Data Vault 2.0 с утилитой для автоматической генерации
SQL-моделей слоёв stage и DV.

---

## Быстрый старт

### 1. Установка зависимостей

```bash
uv sync
```

### 2. Подключение к БД и запуск DBT

Параметры подключения указываются в локальном profiles.yml через флаг --profiles-dir.
Проверка подключения командой debug
```bash
dbt debug --profiles-dir <путь к папке c profiles.yml>
```

### 3. Генерация dbt-модели

Из корня проекта:

```bash
dvgen --target-dir tsm_dwh
```

Команда:
1. Запускает `dbt run-operation dv_generate_source` и сохраняет `tsm_dwh/models/dv/sources.yml`
2. Генерирует SQL-файлы stage-моделей в `tsm_dwh/models/stage/`
3. Генерирует SQL-файлы DV-моделей (hub, link, sat, msat, esat) в `tsm_dwh/models/dv/`


## Рабочий процесс

```
импорт metadata.csv  →  dvgen  →  stage/*.sql + dv/*.sql  →  сборка витрин
```

## Справка по Data Vault 2.0

### Слои

| Слой | Префикс | Описание |
|------|---------|----------|
| **Stage** | `stg_` | Сырые данные из источников. Добавляются системные поля, вычисляются хэш-ключи и хэш-диффы |
| **Data Vault** | `dv_` | Неизменяемое историческое хранилище. Данные только добавляются. Состоит из hub, link, sat, msat, esat |
| **Витрины** | `mart_` | Бизнес-ориентированные модели поверх DV для аналитики: денормализация таблиц, агрегации, расчёт показателей |

---

### Системные поля (все таблицы DV)

| Поле | Тип | Описание |
|------|-----|----------|
| `loadts` | `timestamptz` | Метка времени загрузки записи в хранилище |
| `recsource` | `varchar` | Идентификатор источника данных |
| `hkcode` | `varchar` | Доп. код для хэширования бизнес ключей на случай хэш коллизии |

---

### Виды таблиц Data Vault

#### Hub — `dv_hub_{сущность}`

Уникальный реестр бизнес-объектов. Хранит только бизнес-ключи — без атрибутов и связей.

| Поле | Пример | Описание |
|------|--------|----------|
| `hk_{table}` | `hk_dv_hub_fact_work` | Хэш-ключ (SHA-1 от бизнес-ключа) — PK |
| `bk_{ключ}` | `bk_работа_uuid` | Бизнес-ключ из источника |
| `loadts`, `recsource`, `hkcode` | | Системные поля |

---

#### Link — `dv_lnk_{сущность1}_{сущность2}`

Связи между хабами. Хранит хэш-ключи родительских хабов — без атрибутов.

| Поле | Пример | Описание |
|------|--------|----------|
| `hk_{table}` | `hk_dv_lnk_fact_journal_fact_work` | Хэш-ключ линка (SHA-1 от ключей родителей) — PK |
| `hk_{hub_1}` | `hk_dv_hub_fact_journal` | Хэш-ключ первого родительского хаба — FK |
| `hk_{hub_2}` | `hk_dv_hub_fact_work` | Хэш-ключ второго родительского хаба — FK |
| `loadts`, `recsource`, `hkcode` | | Системные поля |

---

#### Satellite — `dv_sat_{сущность}`

Атрибуты хаба или линка с полной историей изменений. Новая запись добавляется только при изменении хотя бы одного атрибута (контроль через `hdiff`).

| Поле | Пример | Описание |
|------|--------|----------|
| `hk_{parent}` | `hk_dv_hub_fact_work` | Хэш-ключ родительского хаба/линка — FK |
| `hdiff_{table}` | `hdiff_dv_sat_fact_work` | Хэш-дифф (SHA-1 от всех атрибутов) — контроль изменений |
| атрибуты | `объем_работы`, `примечание` | Бизнес-атрибуты |
| `loadts`, `recsource` | | Системные поля |

---

#### Multi-active satellite — `dv_msat_{сущность}`

Спутник для многострочных атрибутов. Каждой строке присваивается `sub_seq` в рамках одной загрузки. Версионируется по всему набору строк `sub_seq` целиком.

| Поле | Пример | Описание |
|------|--------|----------|
| `hk_{parent}` | `hk_dv_hub_fact_work` | Хэш-ключ родительского хаба — FK |
| `hdiff_{table}` | `hdiff_dv_msat_fact_pikets` | Хэш-дифф строки |
| `sub_seq` | | Порядковый номер строки внутри одной загрузки |
| атрибуты | `пикет_с`, `пикет_по` | Атрибуты строки |
| `loadts`, `recsource` | | Системные поля |

---

#### Effectivity satellite — `dv_esat_{линк}`

Хранит периоды активности связи в линке (`start_date` / `end_date`). Позволяет отслеживать, когда связь между объектами существовала.

| Поле | Пример | Описание |
|------|--------|----------|
| `hk_{parent_link}` | `hk_dv_lnk_fact_journal_fact_work` | Хэш-ключ линка — FK |
| `start_date` | | Дата начала активности связи |
| `end_date` | | Дата окончания (`NULL` = связь активна) |
| `loadts` | | Системное поле |
